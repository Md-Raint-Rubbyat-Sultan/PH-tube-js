Question-1: Discuss the scope of var, let, and const?

SL var let const
1 Variables declared with "var" are function-scoped, meaning they are only accessible within the function where they are defined. Variables declared with "let" are block-scoped, which means they are limited in scope to the block (typically enclosed by curly braces {}) in which they are defined. This includes blocks within functions, loops, and conditional statements. Variables declared with "const" are also block-scoped, just like "let". They are limited in scope to the block in which they are defined.
2 "var" declarations are hoisted to the top of their containing function or global scope also known as Hoisting. This means you can use a "var" variable before it's declared in the code, although its value will be "undefined" until the assignment is encountered. Like "var", "let" declarations are hoisted to the top of their containing block but are not initialized. Trying to access a "let" variable before its declaration will result in a "ReferenceError". "const" declarations are hoisted to the top of their containing block but, like "let", are not initialized. Attempting to access a "const" variable before its declaration results in a "ReferenceError".
3 Variables declared with "var" can be reassigned and have their values changed. Variables declared with "let" can be reassigned, so their values can change over time. Variables declared with "const" cannot be reassigned after their initial value assignment. This makes them constant or immutable in the sense that their value cannot change once set.

Question-2: Tell us the use cases of undefined and null?

SL undefined VS null
1 "undefined" represents a variable that has been declared but has not been assigned a value. It is also the default value of function parameters that are not provided when the function is called. "null" represents the intentional absence of any object value or a variable intentionally set to no value or no object.
2 Common use cases for "undefined" include checking if a variable or property exists or if a function argument has been provided. Common use cases for null include clearing or resetting variables or properties, indicating that an object doesn't have a particular value, and explicitly signaling that something is intentionally empty.
3 When a variable is declared but not assigned a value, it's automatically initialized with "undefined". Unlike "undefined", "null" is explicitly set by the programmer to indicate that a variable should have no value or that a property is intentionally empty.

Question-3: What do you mean by REST API?

SL REST API
1 Statelessness: REST APIs are designed to be stateless, which means that each request from a client to the server must contain all the information needed to understand and process the request. There should be no reliance on the server's previous requests or states. Statelessness simplifies server implementation and makes APIs more scalable.
2 Resources: In REST, everything is considered a resource, and each resource is identified by a unique URL (Uniform Resource Locator). Resources can represent data objects, services, or entities in the application. For example, a REST API for a book store might have resources like /books, /authors, and /orders.
3 HTTP Methods: REST APIs use standard HTTP methods (GET, POST, PUT, DELETE, etc.) to perform operations on resources. Each HTTP method has a specific meaning:
  • GET: Retrieve data from a resource.
  • POST: Create a new resource.
  • PUT: Update an existing resource or create it if it doesn't exist.
  • DELETE: Remove a resource.
  • Other methods like PATCH, HEAD, and OPTIONS can also be used for specific purposes.
4 Uniform Interface: REST APIs follow a uniform and consistent interface design, which simplifies client-server communication. This includes using standard HTTP status codes to indicate the result of an operation (e.g., 200 for success, 404 for resource not found, 500 for server error).
5 Representation: Resources in REST are represented in a format that can be easily processed by clients, often in JSON or XML. Clients can request specific representations of resources by specifying the desired media type in the HTTP headers.
6 Stateless Communication: REST is based on stateless communication, meaning that each request from the client to the server should contain all the information needed for processing. This makes it easier to scale the server and handle requests from multiple clients.
7 Layered System: REST allows for the use of a layered system architecture, where different components (e.g., load balancers, caches, security layers) can be added between the client and server without affecting the overall system.
8 Client-Server Separation: REST separates the client and server components, allowing them to evolve independently. This separation simplifies client and server implementations and promotes scalability.
9 REST APIs are commonly used for building web services that can be accessed over the internet, making them a fundamental technology for enabling communication between different software applications, services, and devices. They are used extensively in web and mobile app development, IoT (Internet of Things), and integration between various software systems.
